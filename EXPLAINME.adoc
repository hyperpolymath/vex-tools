== What vex-tools Contains

`vex-tools` is a monorepo consolidating six previously independent repositories:

* `vexometer/` — The Irritation Surface Analyser (ISA): Ada-based tool measuring
  LLM irritation across 10 standardised dimensions
* `vexometer-satellites/` — Umbrella hub for satellite intervention modules
* `satellite-template/` — Scaffolding template for new satellite projects
* `lazy-eliminator/` — Rust tool detecting lazy patterns in LLM-generated code
* `vext/` — Vext verifiable communications protocol (Rust)
* `vext-email-gateway/` — Email bridge for Vext (prototype)

== Claim 1: Consolidated monorepo for all VEX-related tools

[quote, README]
____
Consolidated monorepo for all VEX-related tools centered on The Vexometer:
Irritation Surface Analyser (ISA) for LLMs and related tools, its satellite
intervention system, and the Vext verifiable communications protocol.
____

*How it works.*
The root `Justfile` provides monorepo-wide orchestration: `just build-all`
invokes the Ada GPRbuild for `vexometer/`, `cargo build` for `vext/` and
`lazy-eliminator/`, and scaffolding checks for `satellite-template/`.
`just test-all` runs each sub-project's test suite in sequence.
`just ci-gate` is the single command used in CI to verify the whole monorepo
meets quality and security gates.
`vexometer/src/vexometer-core.ads` defines the 10-dimension `Metric_Category`
enumeration that is the conceptual centre of the toolkit.
`vext/src/vext-core/` implements the cryptographic neutrality proof primitives
that allow Vext messages to carry verifiable provenance.
`lazy-eliminator/src/` detects the completeness failures (stubs, ellipses,
truncated outputs) that `vexometer`'s `Completion_Integrity` metric measures.

*Honest caveat.*
The `vext-email-gateway/` sub-project is explicitly a prototype; the email
bridge is structurally present but end-to-end wiring (SMTP auth, delivery
receipt loop-back to the Vext audit log) is in progress.

== Claim 2: Satellite intervention system with a reusable template

[quote, README]
____
ISA Satellites (vexometer-satellites) — Umbrella documentation and
coordination hub for the satellite intervention system. Satellite Template —
Scaffolding template for creating new ISA satellite projects.
____

*How it works.*
`satellite-template/` provides the canonical directory skeleton for a new
satellite: a `Justfile`, `stapeln.toml`, `contractile.just`, `ROADMAP.adoc`,
`TEST-NEEDS.md`, `PROOF-NEEDS.md`, and stub Ada/Rust source files.
Each satellite is an independent intervention module that consumes ISA scores
from the `vexometer-api` and responds with corrective actions (e.g. prompting
reformulation, flagging sycophancy patterns to the user).
`vexometer-satellites/` maintains the registry of active satellites with
their intervention domains, protocol versions, and status.
`scripts/` contains helpers for generating Vext trust keys (`just trust-generate`)
and verifying signed messages (`just trust-verify`) — demonstrating the
cryptographic layer end-to-end.

*Honest caveat.*
Individual satellite implementations are not in this repository — they are
separate repos that follow the template.
The `vexometer-satellites/` hub is documentation-primary: it coordinates but
does not contain the satellite runtime code.
