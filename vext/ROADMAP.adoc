== Overview

This roadmap outlines the development plan for vext (IRC notification daemon for git operations).

== v1.0 - Production Release (Current)

**Target:** 2026-02
**Status:** Release candidate

=== Core Features
* ✅ IRC notification daemon for git operations
* ✅ Connection pooling and management
* ✅ HTTP webhook receiver
* ✅ GitLab/GitHub webhook support
* ✅ Pandoc integration for document conversion
* ✅ Hunspell spell checking for commit messages
* ✅ Internationalization support (en-US, es-ES, fr-FR)

=== Testing & Quality
* ✅ Unit tests
* ✅ Integration tests
* ✅ ClusterFuzzLite fuzzing
* ✅ Stress tests
* ✅ Benchmarks (criterion)

=== Documentation
* ✅ README.adoc
* ✅ 6 SCM files (STATE, META, ECOSYSTEM, PLAYBOOK, AGENTIC, NEUROSYM)
* ✅ Security policy
* ✅ Usage guide
* ✅ Technology stack documentation

=== Remaining for v1.0
* [ ] Add SPDX headers to all SCM files
* [ ] Add image alt text for accessibility
* [ ] Complete feature documentation in README

== v1.1 - Stability & Polish (Next)

**Target:** 2026-03
**Focus:** Bug fixes, performance improvements, accessibility

=== Enhancements
* Enhanced error messages and debugging
* Performance optimization (connection pooling, message batching)
* Expanded i18n language support
* WCAG AAA accessibility compliance

=== Documentation
* Video tutorials
* Integration examples with popular Git hosting platforms
* Performance tuning guide

== v1.2 - Extended Integrations

**Target:** 2026-04

=== New Integrations
* Gitea webhook support
* Bitbucket webhook support
* Custom webhook format support
* Discord notifications (optional feature)
* Slack notifications (optional feature)

=== Advanced Features
* Message templating system
* Conditional notification rules
* Rate limiting per channel
* Message priority queues

== v2.0 - Major Features

**Target:** 2026-Q3

=== Architecture
* Plugin system for extensibility
* Message transformation pipeline
* Event filtering and routing
* Multi-protocol support (IRC, Matrix, XMPP)

=== Neurosymbolic Features
* Commit message analysis and suggestions
* Automated code review notifications
* Integration with hypatia for rule-based notification routing
* A2ML integration for verified documentation

=== K9-SVC Integration
* Self-validating component contracts
* Must-just-nickel validation
* Contractiles integration

== v3.0 - Distributed & Cloud-Native

**Target:** 2027

=== Cloud Features
* Kubernetes operator
* Multi-instance coordination
* Distributed notification delivery
* High availability setup
* Cloud-native monitoring and observability

=== Advanced AI
* Natural language commit message generation
* Intelligent notification prioritization
* Anomaly detection in git activity
* Automated response generation

== Long-term Vision

* Full gitbot-fleet integration
* Hypatia rule-based orchestration
* Formal verification of notification delivery
* Proof-of-work integration for spam prevention
* Distributed trust network for notifications
